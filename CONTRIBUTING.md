### Reporting Bugs
- Check if the bug has already been reported in [Issues](../../issues)
- Use the bug report template
- Include detailed steps to reproduce
- Attach screenshots or error logs if applicable

### We welcome contributors who can help with:

Backend API design and implementation (Python / FastAPI)
Orchestration logic and workflow execution

Datasource integration and testing
Monitoring, metrics, and health-check systems
Documentation, examples, and reference architectures

### Suggesting Features
- Open an issue with the `enhancement` label
- Clearly describe the feature and its use case
- Explain how it benefits users

### Code Contributions

1. **Fork the repository**
   ```bash
   git clone https://github.com/muhammedali275/ai-orchestrator-studio.git
   cd ai-orchestrator-studio
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow existing code style
   - Add tests for new functionality
   - Update documentation as needed

4. **Test your changes**
   ```bash
   # Backend tests
   cd backend
   pytest
   
   # Frontend tests
   cd frontend
   npm test
   ```

5. **Commit with clear messages**
   ```bash
   git commit -m "feat: add new routing strategy"
   git commit -m "fix: resolve chat streaming issue"
   git commit -m "docs: update API documentation"
   ```

6. **Push and create a Pull Request**
   ```bash
   git push origin feature/your-feature-name
   ```

- Follow Material-UI best practices

### Commit Messages
Follow [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` New features
- `fix:` Bug fixes
- `docs:` Documentation changes
- `style:` Code style/formatting
- `refactor:` Code refactoring
- `test:` Test additions/changes
- `chore:` Maintenance tasks
